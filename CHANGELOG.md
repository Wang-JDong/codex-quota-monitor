# 变更日志

本项目使用语义化版本号，记录用户可见变化。

## [0.2.1] - 2026-07-19

### Fixed

- 为每个可信账号增加 `UserMedia` 媒体时间线补充，并与 `UserTweets` 按帖子 ID 合并，覆盖主时间线漏掉的带媒体引用帖。
- 私有 RSSHub 适配器支持媒体路由并刷新 `UserMedia` Query ID；媒体流短暂失败时保留主时间线结果，不影响本轮健康抓取。
- 增加真实漏报帖回归测试、服务编排测试和媒体路由安全测试。

## [0.2.0] - 2026-07-16

### Added

- 高召回、分层证据分类器：支持 `possible_reset` 低置信度候选，并保留分类器版本、置信度和证据元数据。
- `reprocess-unmatched --days 7 --limit 100` 有界补处理命令，按来源单次抓取并沿用 at-most-once 投递。
- 低置信度橙色飞书卡片，包含证据、北京时间、原文链接和“请确认”提示。

### Fixed

- 识别“额度将在几分钟后恢复”等不依赖固定 reset 句式的可信公告表达。

## [0.1.1] - 2026-07-14

### Added

- 独立的 `banked_available` 分类与“可保存重置次数已发放”飞书文案。
- `reprocess-post` 幂等运维命令，可将已保存的历史未匹配帖子按新规则补处理一次。

### Fixed

- RSSHub 标题被截断时改用 description 中更完整的原创正文，避免漏掉后半段的额度语义。
- 增加 `banked reset` 否定、教程、邀请和周期机制负例，避免把非公告内容误报为已发放。

## [0.1.0] - 2026-07-13

### Added

- 固定监控 `@OpenAI`、`@OpenAIDevs`、`@thsottiaux` 和 `@sama` 的免费 X/RSSHub 抓取链路。
- 面向已完成、正在进行和计划重置的确定性分类器。
- 每来源首次基线静默、SQLite 去重和跨重启状态。
- 飞书签名通知、健康告警/恢复迁移，以及 `delivery-resolve` 人工核对。
- 本机端口 `1200`/`1201` 隔离、384 MiB / 30% CPU 限制和按需 RSSHub 回收。
- preflight/postflight 的 SSH、现有节点服务、unit 哈希和监听者保护。
- 分层产品、架构、运维、安全、测试与 ADR 文档。
